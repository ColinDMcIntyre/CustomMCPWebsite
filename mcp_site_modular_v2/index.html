How the setup works

You are not buying a mystery box. The path is simple: get the files, push to GitHub, deploy on Render, fill in your values, and connect your assistant.

1

Download the package

Get the project files and use the guide to follow the same structure shown on this site.

2

Deploy on Render

Push the project to GitHub, connect the repo in Render, then paste the build command, start command, and environment variables.

3

Choose your auth path

Use static OAuth for full MCP client support, or use a simple bearer token path if you just want a more manual fallback.

4

Keep building later

Once it is running, add new tool modules as needed. The structure is the point: you are buying a base you can keep using.

Start with the main setup first. Once that is working, you can keep expanding with additional integrations and Python tool modules as needed.
